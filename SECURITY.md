# Security Policy

## Supported Versions

This project is pre-1.0. Security fixes are applied to the latest development branch until a formal support policy is published.

## Reporting a Vulnerability

Please report security issues privately to the repository maintainers rather than opening a public issue. If the GitHub repository has private vulnerability reporting enabled, use that flow.

Include:

- affected version or commit;
- a concise description of the issue;
- reproduction steps if available;
- any relevant logs or screenshots with secrets removed.

Do not include private Prometheus exports, Grafana credentials, or production data in reports.
